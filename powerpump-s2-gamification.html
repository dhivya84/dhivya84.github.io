02 — MET Gamification

Using Metabolic Science to Incentivize Physical Activity during EV Charging

← 01 Auth & Privacy Overview
The Challenge

The "Equity in Effort" Problem

A "one point per minute" reward system is unfair—it treats a light walk the same as high-intensity interval training (HIIT). To keep users engaged, the reward system had to be scientifically grounded in actual physical effort.

Implementation

The MET Algorithm

I implemented the Metabolic Equivalent of Task (MET) formula to calculate reward "PowerPoints" $P$. This ensures that rewards are proportional to the physiological energy expended during the charging session.

\[ P = \sum_{i=1}^{n} (MET_i \times weight \times duration_i \times \alpha) \]

Where $\alpha$ is a scaling constant and $n$ represents the total activity sessions recorded during the 30-60 minute charge window. This algorithm allows a user who jogs for 10 minutes to earn significantly more than one who stands for 30 minutes, encouraging higher intensity participation.

Infrastructure

Serverless Scalability

The calculation engine is built on Next.js API routes (Serverless). This architecture was chosen for three reasons:

  • Cold Start Efficiency: Charging hubs have "bursty" traffic patterns (e.g., commute hours). Serverless handles these spikes without requiring fixed server overhead.
  • Event-Driven: Calculations are only triggered when a charging session ends, minimizing idle computing costs.
  • Latency: Data synchronization from the wearable occurs in under 2 seconds, providing the user with immediate dopamine feedback via the dashboard.
